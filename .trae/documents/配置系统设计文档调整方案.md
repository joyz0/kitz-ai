# 配置系统设计文档调整方案

## 问题分析

通过对比 `config-system-design.md` 文档与项目中的实际代码实现，发现以下差异：

### 1. 核心功能差异

| 功能 | 文档描述 | 代码实现 | 状态 |
|------|---------|---------|------|
| 配置加载 | 支持从文件、环境变量、命令行参数等多种来源加载配置 | 只支持从文件加载配置，通过环境变量 KITZ_CONFIG_PATH 指定配置路径 | 部分实现 |
| 配置解析 | 支持 JSON5、YAML、JSON 等多种配置格式 | 只支持 JSON5 格式 | 部分实现 |
| 配置验证 | 支持使用模式验证配置的正确性 | 使用 Zod 实现了配置验证 | 已实现 |
| 环境变量替换 | 支持在配置中使用环境变量 | 实现了 ${VAR_NAME} 格式的环境变量替换 | 已实现 |
| 配置热重载 | 支持在不重启服务的情况下更新配置 | 未实现 | 未实现 |
| 配置监控 | 监控配置的使用情况和变更 | 未实现 | 未实现 |

### 2. 技术特性差异

| 特性 | 文档描述 | 代码实现 | 状态 |
|------|---------|---------|------|
| 类型安全 | 提供类型安全的配置访问 | 使用 TypeScript 类型定义和 Zod Schema | 已实现 |
| 层次结构 | 支持配置的层次结构和继承 | 支持配置的层次结构 | 已实现 |
| 默认值 | 为配置项提供默认值 | 实现了默认值应用功能 | 已实现 |
| 配置合并 | 支持多个配置源的合并 | 实现了通过 $include 指令合并多个配置文件 | 已实现 |
| 配置加密 | 支持敏感配置的加密存储 | 未实现 | 未实现 |

### 3. 架构设计差异

| 组件 | 文档描述 | 代码实现 | 状态 |
|------|---------|---------|------|
| 配置加载器 (Loader) | 负责从各种来源加载配置文件 | 在 io.ts 中实现，负责从文件加载配置 | 部分实现 |
| 配置解析器 (Parser) | 负责解析不同格式的配置文件 | 在 io.ts 中实现，使用 JSON5 解析配置 | 部分实现 |
| 配置验证器 (Validator) | 负责验证配置的正确性和完整性 | 在 schema.ts 中实现，使用 Zod 进行配置验证 | 已实现 |
| 配置管理器 (Manager) | 负责管理配置的生命周期和访问 | 在 io.ts 中实现，负责管理配置的生命周期和访问 | 已实现 |
| 配置监听器 (Watcher) | 负责监控配置的变更 | 未实现 | 未实现 |

### 4. 其他实现细节

- **配置缓存**：代码实现了配置缓存功能，通过 KITZ_CONFIG_CACHE_MS 环境变量控制缓存时间
- **配置备份**：代码在写入配置时会创建备份文件
- **配置审计**：代码中提到了配置审计日志文件名，但没有实现具体的审计功能

## 调整方案

### 1. 更新核心功能部分

- **配置加载**：修改为只支持从文件加载配置，通过环境变量 KITZ_CONFIG_PATH 指定配置路径
- **配置解析**：修改为只支持 JSON5 格式
- **配置热重载**：移除或标记为未实现
- **配置监控**：移除或标记为未实现

### 2. 更新技术特性部分

- **配置加密**：移除或标记为未实现

### 3. 更新架构设计部分

- **配置加载器 (Loader)**：修改为只负责从文件加载配置
- **配置解析器 (Parser)**：修改为只负责解析 JSON5 格式的配置文件
- **配置监听器 (Watcher)**：移除或标记为未实现

### 4. 添加新的实现细节

- **配置缓存**：添加配置缓存功能的描述
- **配置备份**：添加配置备份功能的描述
- **配置审计**：添加配置审计日志的描述

## 预期结果

调整后，`config-system-design.md` 文档将与项目中的实际代码实现保持一致，准确反映配置系统的功能和架构。
