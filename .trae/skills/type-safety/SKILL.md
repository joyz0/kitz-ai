# 类型安全与 Schema 驱动 Skill

## Triggers

### When to use

- 当需要确保系统类型安全时
- 当需要验证配置和输入数据时
- 当需要自动生成 API 文档时
- 当需要确保协议兼容性时

### When NOT to use

- 当系统规模很小，类型安全开销过大时
- 当性能要求极高，无法承受运行时验证开销时
- 当使用非 TypeScript 环境时
- 当不需要严格的类型检查时

## Input

| Parameter     | Type    | Description                | Required |
| ------------- | ------- | -------------------------- | -------- |
| schema        | object  | TypeBox 或 Zod Schema 定义 | true     |
| data          | any     | 需要验证的数据             | true     |
| options       | object  | 验证选项                   | false    |
| generateTypes | boolean | 是否生成 TypeScript 类型   | false    |
| generateDocs  | boolean | 是否生成 API 文档          | false    |

## Output

| Parameter | Type    | Description            |
| --------- | ------- | ---------------------- |
| valid     | boolean | 验证是否通过           |
| data      | any     | 验证后的数据           |
| errors    | array   | 验证错误信息           |
| types     | string  | 生成的 TypeScript 类型 |
| docs      | string  | 生成的 API 文档        |

## Steps

1. **定义 Schema**

   - 使用 TypeBox 定义协议 Schema
   - 使用 Zod 定义配置 Schema
   - 定义基础类型和复合类型

2. **实现验证器**

   - 生成 AJV 验证器
   - 实现运行时验证逻辑
   - 提供错误处理机制

3. **代码生成**

   - 生成 TypeScript 类型定义
   - 生成 API 文档
   - 生成客户端模型

4. **运行时验证**

   - 验证消息数据
   - 验证配置数据
   - 验证输入参数

5. **集成到系统**

   - 与配置系统集成
   - 与网关通信协议集成
   - 与插件系统集成

6. **测试验证**

   - 测试类型定义的正确性
   - 测试验证逻辑
   - 测试类型推断
   - 测试错误处理

7. **优化性能**
   - 批量验证
   - 缓存验证结果
   - 拆分大型 Schema

## Failure Strategy

- **验证失败**：返回详细的错误信息，包含路径和修复建议
- **Schema 定义错误**：在编译时捕获，提供明确的错误信息
- **运行时验证错误**：记录错误日志，返回友好的错误信息
- **性能问题**：优化验证逻辑，使用缓存和批量验证

## 核心理念

**类型即文档，验证即安全**

- TypeBox 定义运行时 Schema
- Zod 用于配置验证
- TypeScript 提供编译时类型
- 自动生成 API 文档

## 架构层次

```
┌─────────────────────────────────────┐
│      Schema 定义层 (schema.ts)       │
│  - TypeBox 类型定义                   │

│  - 方法/事件枚举                     │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│      验证器层 (validator.ts)         │
│  - 生成 AJV 验证器                    │

│  - 生成 JSON Schema                  │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│      代码生成层 (codegen.ts)         │
│  - 生成客户端模型                      │

│  - 生成 TypeScript 接口                │
└─────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│      运行时验证层 (runtime.ts)       │
│  - 消息验证                          │
│  - 配置验证                          │

└─────────────────────────────────────┘
```

## 依赖关系

- **依赖模块**：无
- **被依赖模块**：配置系统、网关通信协议、内存管理系统、插件化架构

## 最佳实践

1. **Schema 单一来源**：TypeBox 同时用于验证和类型推导
2. **严格模式**：`additionalProperties: false` 防止未知字段
3. **错误友好**：验证错误包含精确路径和修复建议
4. **版本化**：协议 Schema 包含版本号
5. **代码生成**：从 Schema 自动生成 Swift/TypeScript 类型

## 性能考量

- **验证开销**：运行时验证会增加延迟
  - 典型开销：每个对象 0.1-1ms
  - 优化：批量验证、缓存验证结果
- **Schema 大小**：过大的 Schema 会影响编译速度
  - 建议：按模块拆分 Schema
  - 避免：单个 Schema 文件超过 1000 行

## 安全考量

- **输入验证**：所有外部输入必须经过 Schema 验证
- **类型安全**：使用 TypeScript 编译时类型检查
- **数据完整性**：验证确保数据符合预期格式
- **错误处理**：验证错误提供明确的修复建议

## 扩展点

- **自定义验证器**：支持自定义验证规则
- **多语言支持**：生成多种语言的类型定义
- **文档生成**：从 Schema 生成 API 文档
- **类型推断**：增强 TypeScript 类型推断能力
