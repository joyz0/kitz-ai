# 会话管理 Skill

## Triggers

### When to use

- 当需要管理用户会话和对话历史时
- 当需要实现会话隔离和状态管理时
- 当需要自动维护会话生命周期时
- 当需要控制会话存储和磁盘使用时

### When NOT to use

- 当系统不需要会话状态时
- 当系统规模很小，会话管理开销过大时
- 当不需要会话隔离和持久化时
- 当性能要求极高，无法承受会话管理开销时

## Input

| Parameter | Type   | Description                 | Required |
| --------- | ------ | --------------------------- | -------- |
| agentId   | string | 代理标识符                  | true     |
| channel   | string | 通道名称                    | true     |
| type      | string | 会话类型 (dm/group/channel) | true     |
| peerId    | string | 对等方标识符                | true     |
| groupId   | string | 群组标识符                  | false    |
| message   | object | 消息对象                    | false    |
| sessionId | string | 会话标识符                  | false    |

## Output

| Parameter | Type    | Description  |
| --------- | ------- | ------------ |
| session   | object  | 会话对象     |
| messages  | array   | 会话消息列表 |
| sessions  | array   | 所有会话列表 |
| success   | boolean | 操作是否成功 |
| error     | string  | 错误信息     |

## Steps

1. **实现会话键管理**

   - 实现会话键生成
   - 定义隔离级别
   - 实现会话分组

2. **实现会话存储**

   - 实现会话索引管理
   - 实现对话记录存储
   - 实现压缩和归档

3. **实现会话维护**

   - 实现会话清理
   - 实现磁盘空间管理
   - 实现自动归档

4. **实现会话 API**

   - 实现会话创建/获取
   - 实现消息添加
   - 实现会话查询

5. **配置会话系统**

   - 配置 DM 会话作用域
   - 配置身份链接
   - 配置重置策略
   - 配置维护策略

6. **测试会话系统**

   - 测试会话键生成
   - 测试会话存储
   - 测试会话维护
   - 测试消息添加和获取

7. **优化会话系统**
   - 优化存储效率
   - 优化检索速度
   - 优化磁盘空间使用

## Failure Strategy

- **会话创建失败**：返回错误信息，记录日志
- **消息添加失败**：返回错误信息，记录日志
- **会话存储失败**：尝试恢复，记录错误，必要时使用备用存储
- **磁盘空间不足**：清理旧会话，发出警告，限制新会话创建
- **会话维护失败**：记录错误，尝试重试，必要时手动干预

## 核心理念

**会话即状态，隔离优先**

- 每个会话有独立的上下文
- 支持按用户/频道/群组隔离
- 自动维护会话生命周期
- 透明的压缩和归档

## 依赖关系

- **依赖模块**：配置系统、日志与可观测性、插件化架构、网关通信协议
- **被依赖模块**：无

## 最佳实践

1. **安全隔离**：多用户场景使用 `per-channel-peer`
2. **身份链接**：跨渠道识别同一用户
3. **自动维护**：生产环境使用 `enforce` 模式
4. **磁盘预算**：设置 `maxDiskBytes` 防止无限增长
5. **归档策略**：保留重要会话，归档旧会话

## 性能考量

- **存储效率**：会话存储的性能和空间使用
  - 优化：使用压缩和归档
  - 避免：无限制的会话增长
- **检索速度**：会话和消息的检索速度
  - 建议：使用索引和缓存
  - 避免：全量扫描

## 安全考量

- **数据保护**：会话数据的安全存储
- **隐私保护**：用户消息的隐私保护
- **访问控制**：会话访问的权限控制
- **数据备份**：定期备份会话数据
