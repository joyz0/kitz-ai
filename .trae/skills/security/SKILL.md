# 安全与认证 Skill

## Triggers

### When to use
- 当需要保护系统免受未授权访问时
- 当需要实现设备配对和认证机制时
- 当需要控制对通道和功能的访问权限时
- 当需要监控和防御安全威胁时

### When NOT to use
- 当系统运行在完全隔离的环境中，不需要外部访问时
- 当开发测试环境，安全要求较低时
- 当性能要求极高，无法承受认证开销时
- 当系统规模很小，安全风险可接受时

## Input

| Parameter | Type | Description | Required |
|-----------|------|-------------|----------|
| deviceId | string | 设备唯一标识符 | true |
| publicKey | string | 设备公钥 | true |
| pairingCode | string | 配对码 | false |
| token | string | 设备令牌 | false |
| signature | string | 对挑战的签名 | false |
| channel | string | 通道名称 | false |
| from | string | 来源标识符 | false |
| ip | string | 客户端 IP 地址 | false |

## Output

| Parameter | Type | Description |
|-----------|------|-------------|
| token | string | 设备令牌 |
| isValid | boolean | 认证是否成功 |
| hasAccess | boolean | 是否有访问权限 |
| securityEvents | array | 安全事件列表 |
| rateLimitAllowed | boolean | 是否允许请求（速率限制） |

## Steps

1. **实现认证系统**
   - 实现设备配对机制
   - 实现连接认证逻辑
   - 实现令牌管理

2. **实现访问控制**
   - 实现 DM 访问策略
   - 实现通道权限管理
   - 实现插件权限控制

3. **实现安全审计**
   - 实现安全事件记录
   - 实现权限变更审计
   - 实现异常行为检测

4. **实现威胁防护**
   - 实现 DDoS 防护
   - 实现重放攻击防护
   - 实现中间人攻击防护

5. **配置安全系统**
   - 配置配对设置
   - 配置访问控制策略
   - 配置威胁防护参数

6. **测试安全系统**
   - 测试认证流程
   - 测试访问控制策略
   - 测试安全审计
   - 测试威胁防护

7. **安全响应**
   - 实现安全事件响应流程
   - 实现漏洞修复机制
   - 实现安全公告发布

## Failure Strategy

- **认证失败**：拒绝访问，记录安全事件
- **配对失败**：返回配对码，要求用户确认
- **访问被拒绝**：返回权限错误，记录访问尝试
- **安全威胁检测**：采取相应防护措施，记录威胁事件
- **密钥泄露**：立即吊销凭据，生成新凭据

## 核心理念

**零信任，最小权限**

- 所有连接需要认证
- 设备配对机制
- 本地信任自动批准
- 远程连接显式授权

## 依赖关系

- **依赖模块**：配置系统、日志与可观测性
- **被依赖模块**：CLI系统、网关通信协议、插件化架构

## 最佳实践

1. **配对优先**：默认使用 pairing 策略
2. **本地信任**：本地连接自动批准（可选）
3. **签名验证**：所有连接请求必须签名
4. **令牌过期**：设备令牌设置过期时间
5. **审计日志**：记录所有配对和认证事件

## 性能考量

- **认证开销**：认证过程会增加连接延迟
  - 优化：缓存认证结果
  - 避免：频繁的认证验证
- **加密性能**：加密操作会消耗 CPU 资源
  - 建议：使用硬件加速的加密算法
  - 避免：使用过于复杂的加密算法

## 安全考量

- **密钥管理**：安全存储密钥和令牌
- **证书验证**：验证所有 TLS 证书
- **权限分离**：不同功能使用不同的权限
- **安全更新**：及时更新依赖和安全补丁

## 扩展点

- **多因素认证**：支持 2FA 等多因素认证
- **单点登录**：集成 SSO 系统
- **安全扫描**：定期安全扫描和漏洞检测
- **安全监控**：实时安全监控和告警
